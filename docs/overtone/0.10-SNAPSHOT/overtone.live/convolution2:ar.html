convolution2:ar

[in :none, kernel :none, trigger :none, framesize 512.0] in - Processing target kernel - Buffer index for the fixed kernel, may be modulated in combination with the trigger trigger - Update the kernel on a change from <= 0 to > 0 framesize - Size of FFT frame, must be a power of two. Convolution uses twice this number internally, maximum value you can give this argument is 2^16 = 65536. Note that it gets progressively more expensive to run for higher powers! 512, 1024, 2048, 4096 standard. Strict convolution with fixed kernel which can be updated using a trigger signal. See Steven W Smith, The Scientist and Engineer's Guide to Digital Signal Processing: chapter 18: http:// www.dspguide.com/ch18.htm Categories: FFT Rates: [ :ar ] Default rate: :ar
